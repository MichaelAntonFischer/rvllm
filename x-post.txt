what changed since last post
- removed ALL f32 from the data path. weights, hidden states, embeddings, norms, KV cache -- everything is f16. f32 only remains for RoPE trig tables and the final argmax logits boundary.
- found and fixed the coherence bug. fused QKV GEMM outputs [T, QKV_dim] (interleaved per token) but the split assumed [all_Q | all_K | all_V] layout. identical for T=1 (decode), completely wrong for T>1 (prefill). corrupted the KV cache on every prompt. fix: unfused projections for prefill, fused for decode.
- rms_norm_eps was hardcoded at 1e-5 instead of reading from model config (Qwen2.5 uses 1e-6). now read from config.json and logged at startup.
- pre-allocated metadata + output GPU buffers at max size so CUDA graph-captured pointers never go stale on reallocation.
- removed ~530 lines of dead f32 CPU attention fallback code. removed duplicate f32 weight storage (was loading weights twice -- once f32, once f16).
architecture
- pure f16 forward: embed(f16) -> 28 layers(f16) -> norm(f16) -> LM head(f16xf16->f32) -> argmax
- fused QKV + gate+up weights for decode (T=1): 5 GEMMs -> 2 per layer
- unfused for prefill (T>1): correct [all_Q | all_K | all_V] layout
- cross-layer residual+RMSNorm fusion
- CUDA graph capture per exact batch size, 1 cuGraphLaunch replaces 284 kernel launches
- custom GEMV (M=1) + cublasLt split-K (M<=32) + cuBLAS hgemm (M>32)
- FlashAttention-2 decode with f16 I/O, f16 KV cache
- dedicated GPU OS thread, async DtoH with pinned memory
- 10 architectures: Llama, Mistral, Qwen2, Phi, Gemma, Mixtral, DeepSeek, GPT-NeoX, StableLM, Cohere
